# 视觉海岸批量处理工作台 · 打包与安装指南

把编译好的程序做成「可分发 / 可安装」的形式，让别人在**全新的 Windows 电脑**上直接使用。

---

## 0. 先看结论：这个程序依赖什么？

用 `objdump` 实测 `target\release\visual_coast_batch.exe` 的导入表，它**只依赖 Windows 自带的系统 DLL**：

```
kernel32 / user32 / gdi32 / opengl32 / shell32 / ole32 / combase /
crypt32 / secur32 / bcryptprimitives   (TLS 走系统 Schannel)
uiautomationcore / dwmapi / uxtheme / imm32 / ws2_32 …
api-ms-win-crt-*                       (Universal CRT，Win10/11 自带)
```

**不需要**：
- ❌ Visual C++ 运行库（无 `vcruntime140.dll`）
- ❌ MinGW 运行时（`libgcc_s_seh-1.dll` / `libstdc++-6.dll` / `libwinpthread-1.dll` 已**静态链接**）
- ❌ .NET / WebView2 / OpenSSL

> ✅ 含义：在 **Windows 10 / 11（64 位）** 上，把那个 9MB 的 exe 单独拷过去**双击就能运行**，无需装任何运行时。
> ⚠️ 仅 Windows 7/8.1 可能缺 Universal CRT，需先打「KB2999226」更新（目标用户基本都是 Win10/11，可忽略）。

## 1. 前置：先编出发布版 exe

在项目根目录执行（脚本会自动配置 GNU 工具链 PATH）：

```powershell
powershell -ExecutionPolicy Bypass -File build.ps1 -Release
```

产物（两个文件内容相同，一个英文名一个中文名）：

```
target\release\visual_coast_batch.exe
target\release\视觉海岸批量处理工作台.exe        ← 分发用这个
```

后面所有打包方式都基于这个 release exe。

---

## 2. ⚠️ 一个必须知道的细节：配置文件写在 exe 同目录

程序把参数/预设保存在 **exe 所在目录**的 `vc_batch_config.json`。

这带来一个**安装位置的硬性要求**：

- ❌ **不要装到 `C:\Program Files\…`**。普通用户对该目录**没有写权限**，会导致「参数设置保存失败、下次打开记不住」。
- ✅ **装到用户可写目录**，例如 `%LOCALAPPDATA%\Programs\…`（本指南的安装脚本就是这么做的），这样配置能正常保存，且**无需管理员权限、不弹 UAC**。

（输出图片、`_manifest.csv` 写到你在界面里选的「输出文件夹」，不受影响。）

## 3. 方式一：绿色便携版（最简单，零安装）

适合「直接发给朋友、U 盘拷贝」。

1. 新建一个文件夹，例如 `视觉海岸批量处理工作台_v1.0.0\`。
2. 把 `target\release\视觉海岸批量处理工作台.exe` 放进去。
3. （可选）把示例工作流 `Flux2-Klein人像精修_高清放大_保留光影.json` 一起放进去，方便对方自动识别节点。
4. （可选）放一个 `使用说明.txt`，写上：双击 exe → 填 API Key / 工作流 ID / 输入输出文件夹 → 开始。
5. 整个文件夹右键「压缩为 ZIP」→ 得到 `视觉海岸批量处理工作台_v1.0.0.zip`。

对方拿到后：解压到**任意可写目录**（如桌面、`D:\软件\`，**别放 Program Files**），双击 exe 即用。

> 优点：零门槛。缺点：没有开始菜单/卸载项，谈不上「安装」。要正式安装包看方式二。

## 4. 方式二：Inno Setup 安装包（推荐）

生成一个专业的 `setup.exe`：带安装向导、开始菜单、桌面快捷方式、卸载程序。免费、简单。

### 4.1 安装 Inno Setup

1. 到官网下载并安装：https://jrsoftware.org/isdl.php （选 `innosetup-6.x.x.exe`，6.3 以上）。
2. 安装时勾选 Inno Setup Preprocessor（默认即可）。

### 4.2 （可选）准备简体中文向导语言

Inno 官方安装包**不自带简体中文**。如果想要中文安装向导：
- 到 https://github.com/jrsoftware/issrc/tree/main/Files/Languages/Unofficial 下载 `ChineseSimplified.isl`，
- 放到 Inno 安装目录的 `Languages\` 子目录下（如 `C:\Program Files (x86)\Inno Setup 6\Languages\`）。

不弄也行——下面脚本默认用英文向导，程序本身仍是中文界面。

### 4.3 （可选）给 exe 加图标

见本文第 6 节。没有图标也能打包，只是用系统默认图标。

### 4.4 编写安装脚本 `installer.iss`

在**项目根目录**新建文件 `installer.iss`，内容如下（可直接复制）：

```iss
; 视觉海岸批量处理工作台 —— Inno Setup 安装脚本
; 在 Inno Setup 里打开本文件，点 Build（或用命令行 ISCC.exe 编译）。

#define MyAppName "视觉海岸批量处理工作台"
#define MyAppVersion "1.0.0"
#define MyAppPublisher "你的名字或团队"
#define MyAppExe "视觉海岸批量处理工作台.exe"

[Setup]
AppId={{8F3A2C71-9B4D-4E6A-AC10-VISUALCOAST01}}
AppName={#MyAppName}
AppVersion={#MyAppVersion}
AppPublisher={#MyAppPublisher}

; 关键：装到用户可写目录（LocalAppData），保证配置文件能保存、且免管理员权限
DefaultDirName={localappdata}\Programs\VisualCoastBatch
DisableProgramGroupPage=yes
PrivilegesRequired=lowest
PrivilegesRequiredOverridesAllowed=dialog

; 只允许 64 位 Windows（含 ARM64 的 x64 仿真）
ArchitecturesAllowed=x64compatible
ArchitecturesInstallIn64BitMode=x64compatible

OutputDir=installer
OutputBaseFilename=视觉海岸批量处理工作台_安装程序_v{#MyAppVersion}
Compression=lzma2/max
SolidCompression=yes
WizardStyle=modern

; 有图标就启用下面两行（assets\app.ico 需存在）
;SetupIconFile=assets\app.ico
UninstallDisplayIcon={app}\{#MyAppExe}

[Languages]
; 默认英文向导；放好 ChineseSimplified.isl 后可改成下面注释里的中文行
Name: "default"; MessagesFile: "compiler:Default.isl"
;Name: "default"; MessagesFile: "compiler:Languages\ChineseSimplified.isl"

[Tasks]
Name: "desktopicon"; Description: "创建桌面快捷方式"; GroupDescription: "附加快捷方式:"

[Files]
Source: "target\release\{#MyAppExe}"; DestDir: "{app}"; Flags: ignoreversion
; 可选：把示例工作流一起装上（取消下一行注释）
;Source: "Flux2-Klein人像精修_高清放大_保留光影.json"; DestDir: "{app}"; Flags: ignoreversion

[Icons]
Name: "{autoprograms}\{#MyAppName}"; Filename: "{app}\{#MyAppExe}"
Name: "{autodesktop}\{#MyAppName}"; Filename: "{app}\{#MyAppExe}"; Tasks: desktopicon

[Run]
Filename: "{app}\{#MyAppExe}"; Description: "立即运行 {#MyAppName}"; Flags: nowait postinstall skipifsilent

[UninstallDelete]
; 卸载时清掉运行时生成的配置文件
Type: files; Name: "{app}\vc_batch_config.json"
```

### 4.5 编译生成安装包

两种方式任选其一：

- **图形界面**：用 Inno Setup 打开 `installer.iss` → 菜单 `Build → Compile`（或按 F9）。
- **命令行**：
  ```powershell
  & "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
  ```

成功后得到安装包：

```
installer\视觉海岸批量处理工作台_安装程序_v1.0.0.exe
```

把这个 `setup.exe` 发给别人即可。

### 4.6 安装 / 卸载体验

- 双击 setup.exe → 向导 → 选「创建桌面快捷方式」→ 安装（**不弹 UAC**，因为装到当前用户目录）。
- 开始菜单、桌面会出现「视觉海岸批量处理工作台」。
- 卸载：设置 → 应用 → 找到它 → 卸载（或开始菜单里的卸载项）。

### 4.7 以后重新生成安装包（一行命令）

`installer.iss` 已存在的前提下，改完代码/版本号后，在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File build.ps1 -Release; & "D:\Inno Setup 6\ISCC.exe" installer.iss
```

（先重编 release exe，再用 ISCC 重新打包；只改版本号时先编辑 `installer.iss` 顶部的 `MyAppVersion`。产物仍在 `installer\` 目录。）

## 5. 方式三：MSI 安装包（cargo-wix，适合企业/批量部署）

需要 MSI（域内 GPO 推送、SCCM 等）时用。

1. 安装 WiX Toolset v3：https://github.com/wixtoolset/wix3/releases
2. 安装 cargo 子命令：
   ```powershell
   cargo install cargo-wix
   ```
3. 初始化 + 打包（注意 GNU 工具链 PATH，建议在配置好 PATH 的终端里跑，或先执行 build.ps1 设置环境）：
   ```powershell
   cargo wix init
   cargo wix
   ```
4. 产物在 `target\wix\*.msi`。

> 说明：cargo-wix 默认按 MSI 惯例装到 `Program Files`。由于本程序配置写在 exe 同目录（见第 2 节），用 MSI 时建议改装到用户目录，或后续把配置改存到 `%APPDATA%`（需改代码）。**个人/小团队分发优先用方式二（Inno Setup）更省心。**

---

## 6. 给 exe 加程序图标（可选，但推荐）

有图标的 exe 和安装包更专业。

1. 准备一张方形 PNG（建议 256×256），用工具转成 `.ico`：
   - 在线：https://convertico.com 或 https://icoconvert.com
   - 或用 ImageMagick：`magick convert app.png -define icon:auto-resize=256,128,64,48,32,16 app.ico`
2. 把生成的图标放到项目的 `assets\app.ico`。
3. 重新编译：`powershell -ExecutionPolicy Bypass -File build.ps1 -Release`
   （`build.rs` 检测到 `assets\app.ico` 会自动嵌入到 exe。）
4. 想让安装向导也用这个图标：在 `installer.iss` 里取消 `SetupIconFile=assets\app.ico` 那行的注释。

## 7. 关于 SmartScreen / 杀毒误报（重要心理预期）

- **首次运行/安装，未签名的 exe 会触发 SmartScreen**：弹「Windows 已保护你的电脑」。
  - 手动放行：点「更多信息 → 仍要运行」。
  - 这是所有未做**代码签名**的小软件的通病，不是程序有问题。
- **彻底消除警告**需要购买**代码签名证书**（OV 证书一年约几百到上千元，EV 证书可即时免 SmartScreen），用 `signtool` 给 exe 和 setup.exe 签名：
  ```powershell
  signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 "视觉海岸批量处理工作台.exe"
  ```
  个人分发一般不必买；告诉对方「点更多信息→仍要运行」即可。
- 个别杀毒软件可能对未签名的 Rust/打包 exe 误报，签名同样能缓解。

## 8. 在「全新 Windows」上的验收清单

拿一台（或虚拟机）干净的 Win10/11，**不装任何运行时**，验证：

- [ ] 双击 setup.exe，能正常走完安装向导（无 UAC 弹窗）。
- [ ] 桌面/开始菜单出现快捷方式，点击能打开，**界面中文不乱码**。
- [ ] 填好参数点「开始」，能正常上传/处理/下载（需有效 API Key + 网络）。
- [ ] 关掉再打开，**上次填的参数还在**（验证配置可写——这就是第 2 节为什么不装 Program Files）。
- [ ] 设置 → 应用里能找到并成功卸载。
- [ ] 若个别老旧显卡花屏/打不开窗口 → 见第 9 节。

## 9. 常见问题

| 现象 | 原因 | 处理 |
|---|---|---|
| 双击没反应 / 杀毒拦截 | SmartScreen 或杀软误报未签名程序 | 「更多信息→仍要运行」；或做代码签名（第 7 节） |
| 设置保存不了、重开记不住 | 装到了 `Program Files` 等只读目录 | 改装到用户目录（本指南脚本已是 LocalAppData） |
| 中文显示成 □□□ | 系统缺中文字体（极少见） | 程序会自动找微软雅黑/黑体/宋体；确认系统装了中文字体 |
| 窗口花屏 / 打不开 | 老旧显卡与 wgpu(DX12) 不合 | 让 eframe 改用 glow(OpenGL) 渲染后重编（给 eframe 开 `glow` feature 并在 NativeOptions 设 `renderer: Glow`） |
| Win7/8.1 报缺 DLL | 缺 Universal CRT | 安装微软「KB2999226」更新 |

## 10. 一句话总结

- **快速分发** → 方式一：把 `target\release\视觉海岸批量处理工作台.exe` 压成 zip 发出去（解压即用，别放 Program Files）。
- **正式安装包** → 方式二：装 Inno Setup，用本指南的 `installer.iss` 一键编出 `setup.exe`（推荐）。
- 这个 exe 在 Win10/11 上**自包含、零运行时依赖**，打包时**唯一要注意**的就是：**装到用户可写目录**（因为配置写在 exe 同目录）。
