# 景别判定与 QC 手册（Workflow C 配套 · 给判图的人/Claude Code 用）

> 用途：① 给 `c_batch.py set --shot` 提供**统一的景别判定线索**（谁判都一样）；② 给建册时的**排片规则**；③ 给 `qc` 后的 **2C 评分单**。判定只看**人物原片**本身。

## 一、景别判定线索（四档，对应 --shot）

| --shot | 画面下缘落在哪 | 典型特征 | 易混提醒 |
|---|---|---|---|
| `full` 全景/全身 | 鞋/脚踝可见，或裙摆及地完整入画 | 人物完整、四周仍有环境余量 | 裙摆遮脚也算 full，只要裙摆下缘完整 |
| `medium` 中景 | 膝盖 ~ 腰之间 | 上半身为主、脚不入镜 | **七分身（大腿中部）归 medium**；坐姿按"可见范围"判 |
| `close` 近景 | 胸口 ~ 腋下 | 头肩胸占满画面 | 捧花在画内不影响判档 |
| `closeup` 特写/大特写 | 肩颈以上 | 脸占画面比例很大 | 只露脸+单肩也算 closeup |

补充判定：
- **subjects（人数）**：画里有几位主体就填几（1 或 2）。双人景别**以"更全的那位"为准**（一人全身一人半身 → full）。
- 拿不准卡在两档之间 → **取更"远"的那档**（如 full/medium 之间取 full），因为模型更容易擅自推近、不容易擅自拉远。

## 二、每档候选数（写进 set --candidates，不填用默认）

**默认每单 1 张**（2026-06-12 起删除"出满3候选挑一"的旧纪律）；QC 不过关就重出，要多张挑时再 `set --candidates N`。closeup 脸部特写五官漂移最高发：漂了重出或临时加候选；连续两轮 closeup 都脸漂 → 降档成 close 再试（脸占比小一点，漂移立刻可控）。

## 三、相册级排片规则（init 之后、prompt 之前做）

1. **全册一个基调、一个对话**：`set --key` 只定一次（warm 或 overcast），`set --url` 固化该客户专属对话——这两个就是"整本像同一次拍摄"的底盘。
2. **场景排重优先**：每张单子尽量分配不同 `scene`；目录不够用必须复用时，**复用的两单必须是不同景别**，且成册排版时隔开（c_batch 在 set 时会对重复场景给出 ⚠️ 提醒）。
3. **景别配比参考**（一本 8~12 张）：full 2~3 / medium 3~4 / close 2~3 / closeup 1~2。原片本身的景别分布不满足时，配比让位于"原片是什么景别就出什么"——**绝不为配比改原片景别**（锁景别是②号铁条）。
4. 全景优先分配 `has_foreground_path=true` 的场景（站位可信）；特写分配花海/密花类（虚化好看）——`match_scene.py` 的自动排序就是这个逻辑，手动指定时也照此。

## 四、QC 2C 评分单（每张候选逐项 ✓/✗，9 项全✓才可 select）

| # | 检查项 | ✓ 标准 |
|---|---|---|
| 1 | 保本人 | 脸型/五官结构/气质一眼是同一人（双人=两位都查） |
| 2 | 姿势 | 肢体关系、朝向与原片一致 |
| 3 | 服装款式+配饰 | 不改款不改色不改廓形；捧花/头纱/手套/领结形态在 |
| 4 | 景别占比 | 可见范围与原片相同——没被推近/拉远/补全身体 |
| 5 | 入景接地 | 有合理地面接触+接地阴影；无悬浮/边缘发光/抠图边/贴图感 |
| 6 | 基调统一 | 光向/色温/反差符合全册基调（warm 或 overcast），无串色温 |
| 7 | 双人关系 | （双人单）两人都在、相对位置与互动没变 |
| 8 | 手部/肢体 | 手指数量形态正常、无肢体变形 |
| 9 | 干净 | 无文字/水印/logo、无明显 AI 渲染味 |

## 五、常见失败 → 对策速查

| 症状 | 对策 |
|---|---|
| 景别被拉远/推近（4✗） | 重出；顽固则在该单提示词**末尾**追加一句"务必保持与原片完全相同的取景范围，不要拉远、不要补全身体"再 run |
| 脸漂（1✗） | 重 run 重出（要多张挑就 `--candidates N` 加量）；closeup 连漂两轮 → 降档 close |
| 接地假/悬浮（5✗） | 检查③是否完整；全景确认所配场景 `has_foreground_path=true` |
| 色温串/光向乱（6✗） | 查该场景 `comp_prompt` 是否**残留天气/光照句**（建目录时该删没删干净），删净重新 prompt |
| 双人丢一人/合体（7✗） | 确认 subjects=2 已填；重出，必要时在词里点明"画面中必须同时出现两位人物" |
| 不够 2K | 不算废片——select 会自动标 needs_upscale，collect 后统一 Topaz |

## 六、防呆三条

1. **带第三方水印的场景缩略图，永远不要当参考图上传**——C 本来就只传人物一张、场景走文字，坚持这个形态。
2. 一次只跑一单（run 默认 --max 1），**出完即 QC**，别攒。
3. 改《人物入景》模板前先和负责人达成共识（铁律5）；manifest 里每单记了 `tpl_md5`，改词后效果变化可回溯到版本。

# （v2.1 自动匹配框架 · 看图判定 / 排片 / QC 增补）

## 七、看图判 person_attrs（每张人物原片，喂给 `c_batch set --no NN --attrs '{...}'`）
判定只看人物原片本身；不确定的项标 `confidence:low`（排片速览会高亮让你复核）。
| 字段 | 取值 | 怎么判 |
|---|---|---|
| `shot` | full/medium/close/closeup | 见上一节景别线索 |
| `subjects` | 1 / 2 | 画里几位主体 |
| `facing` | left/right/front/back | 人物（脸/身）主朝向 |
| `open_space` | left/right/none | 画面留白/空旷在哪侧（决定花径从哪进） |
| `has_train` | true/false | 有无长拖尾 |
| `pose_density` | full/loose | 人物/动作占满画面 or 有留白 |
| `view_angle` | eye/high/low | 原片机位平视/俯/仰 |
| `mood` | fresh/rich/dreamy | 气质调性 |
> 例：`set <客户> --no 03 --attrs '{"facing":"left","open_space":"right","has_train":true,"view_angle":"eye","confidence":{"open_space":"low"}}'`

## 八、匹配 + 排片（最配优先、铺排兜底）
1. **最配优先**：在选定 `series` 内，按 person_attrs ↔ 场景标签挑**最契合**的 plate（朝向↔lead_dir、has_train↔supports_train、机位/ground_amount↔能否放住该景别、backdrop/mood 契合）。**别为"不重样"牺牲最配。**
2. **排片兜底（次级）**：逐张选完过一道——同一 plate 复用上限、景别配比对照（一本 8~12：full 2~3/medium 3~4/close 2~3/closeup 1~2）、`lead_dir` 左右**交替有节奏**。
3. **配不到就明说**：某 series 某景别无合适 plate → 报 coverage gap + 降级建议（就近景别/转人工），**绝不闷头挑烂的**。
4. 用 `c_batch plan <客户>` 出**排片速览**（写进 manifest 的 plan 字段，单一真源）；要改就 `set` 改字段再 `plan` 刷新；满意才 `prompt → run`。

## 九、QC 增补两项（接在 2C 九项后）
| # | 检查项 | ✓ 标准 |
|---|---|---|
| 10 | **比例·透视可信** | 视平线/机位与场景一致、人物有合理接地（非悬浮）、占比符合景别、近大远小自然——**像真相机一次拍成**（第一目的） |
| 11 | **人物在空侧无冲突** | 人物没撞在引导线上、没朝画外、放在留白/空侧，与花径走向不打架 |
> 这两项是 v2.1 新控制杆（figure_anchor/比例块）的验收口；新加的杆子必须有人验。
