# 视觉海岸批量处理工作台 · 使用文档

> 给操作同事看的「拿到就能用」说明 + 给分发方看的「怎么打包发出去」。
> 软件是一个 Windows 桌面程序，顶部可在两个模式间切换：
> - **RunningHub 精修 / 高清放大**：把客户原片批量跑 RunningHub 工作流（精修/放大）。
> - **人物入景**：把真人原片放进固定场景库，AI 自动判景别→选景→出成套写实婚纱样片（走 4sapi `gpt-image-2`）。

---

## 一、5 分钟快速上手（人物入景）

1. 双击 **`视觉海岸批量处理工作台.exe`**。
   - 首次可能弹「Windows 已保护你的电脑」→ 点 **「更多信息」→「仍要运行」**（未签名小程序的通病，不是病毒）。
2. 顶部点 **「人物入景」**。此时上方应显示 **「✓ 已加载 18 张 plate」**（4sapi Key、访问令牌、用户ID 也会自动填好）。
3. 选 **调色**（自然 / 暖 / 冷）和 **场景池**（如 `jiaoshi` 礁石 / `bbsh` 步步生花 / 全部）。
4. **选原片素材**（三选一，可叠加）：点 **「文件夹…」** 选整个客户文件夹批量；或点 **「选图…」** 单选/多选几张；或**直接把图片/文件夹拖进窗口**。再点 **「输出文件夹」浏览…** 选结果存放处。
   - 「文件夹」适合整批客户；「选图/拖拽」适合只跑某几张、补单张、或临时挑几张试效果。
5. 设 **并发数**（基础 2–4 即可）、确认 **「跳过已处理（断点续跑）」** 勾着 → 点 **「▶ 开始」**。

软件会自动：**逐张 AI 判景别 → 自动选景 → 装配提示词 → 双图出图**，进度条实时走，结果落到输出夹（文件名 `原片名_c.png`）。
中途可 **「■ 停止」**；再点开始会**跳过已出的**、只补没出的。

> 出完后：点左侧某张单子看 **原片 ↔ 结果** 对比，满意点 **「✓ 选用」**、不行点 **「✗ 判废」**（改下景别再点开始重出）。全部选好点 **「归集已选」**，精选片会归档到 `输出文件夹\final\`。

## 二、软件包里有什么 · 放哪里

拿到的压缩包解压后只有**三个文件**（场景库**不再随包**，改放局域网共享，见第四节）：

```
视觉海岸批量处理工作台\
├── 视觉海岸批量处理工作台.exe     ← 主程序
├── key.txt                        ← 4sapi 凭据（出图/判景别/查额度都靠它）
└── 使用文档.md                    ← 本文档
```

> 场景库（实拍 plate + 标签）放在**局域网共享** `\\DESKTOP-66773HC\ai_work\wedding_scene_lib`，主机统一维护、全员共用；程序**默认就从这里读**，所以包里不再带、也不会越打越大。**前提：你的电脑能访问这个共享**（同一局域网、有读取权限）。

**放在哪里：**
- ✅ 解压到**有写权限的目录**：桌面、`D:\软件\`、`%LOCALAPPDATA%\Programs\` 等。
- ❌ **不要放 `C:\Program Files\`**（普通用户对它没写权限，程序记不住设置）。
- `key.txt` 要和 exe 放**同一目录**（程序在 exe 旁边找凭据）。
- 能上局域网、访问得到 `\\DESKTOP-66773HC\ai_work` 共享即可（场景库在那里）。

在 **Win10 / 11（64 位）** 上无需安装任何运行时，双击即用。

## 三、凭据 key.txt 怎么配

`key.txt` 是纯文本，**三行**，程序启动时自动读取：

```
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx          ← 第1行：出图/判景别用的 sk- 密钥
系统令牌：xxxxxxxxxxxxxxxxxxxxxxxx            ← 第2行：查额度用的系统访问令牌
用户ID：142788                               ← 第3行：查额度用的用户 ID（4sapi 强制要）
```

- **第 1 行 `sk-` 密钥**：4sapi 控制台「令牌」页创建；管**出图（gpt-image-2）**和**判景别（claude-sonnet-4-6）**。
- **第 2 行 系统令牌 / 第 3 行 用户 ID**：4sapi **个人设置 / 系统访问令牌**页获取（页面也会显示你的数字 ID）；**只用于显示余额**，不影响出图。可留空（留空则不显示额度）。

**共用还是各自一份？**
- 交接约定是**共用令牌**（余额共享）→ 直接用分发包里的 `key.txt` 即可，但**要有人盯余额、及时充**。
- 同事若有自己的 4sapi 账号 → 用自己的三行覆盖 `key.txt`。

> 改完 `key.txt` 重启程序生效；也可在界面里直接粘贴（界面填的优先于文件，但不持久化）。
> ⚠️ `key.txt` 含密钥，**只发给可信的人**，别上传公开网盘/群。

## 四、场景库怎么配

「场景库」= 出图用的**固定实拍场景图（plate）+ 标签目录**，是「人物入景」的素材池。**现在统一放局域网共享，主机维护、全员共用、不随程序打包**（避免越打越大）。

### 4.1 共享路径（默认就能用）
```
\\DESKTOP-66773HC\ai_work\wedding_scene_lib\
├── templates\人物入景_双图暖金版.md
└── scenes\  (catalog.json + series_profiles.json + 若干张 plate .jpg)
```
程序**默认就指向这个共享路径**，第一次切到「人物入景」会显示「✓ 已加载 N 张 plate」就对了。界面「场景库」那一行就是它，**你改过会被记住**（下次自动用上次的路径）。

### 4.2 加载不出来 / 想换路径
- 界面「场景库」→ 直接把路径改成共享地址（或 **浏览…** 选到含 `templates\` 和 `scenes\` 的目录）→ 点 **「加载」**。
- 若显示「⚠ 未加载」：多半是**访问不到共享**——检查是否在同一局域网、能否在文件资源管理器打开 `\\DESKTOP-66773HC\ai_work`（需要时输主机账号密码）。

### 4.3 扩库 / 加新场景（**只在共享上改一次，全员立即生效**）
四轴解耦设计：**加场景只动数据、不改程序**。在共享 `wedding_scene_lib\` 上操作：
1. 把**干净无水印**的场景图（.jpg）放进 `scenes\`。
2. 在 `scenes\catalog.json` 的 `scenes` 数组里加一条，关键字段：`id` / `file` / `series`（概念池名，如 `bbsh`/`jiaoshi`）/ `fits_shot`（适配景别 `full`/`medium`/`close`/`closeup`，可多个）/ `composition` / `has_foreground_path` / `figure_anchor`（站位锚点）/ `comp_prompt`（构图描述，**只写构图、删掉光照天气句**）。
3. 全新概念池（如海边 `beach`）再在 `series_profiles.json` 复制一条配置卡、把「场景专属话」换成新概念说法。
4. 各台程序回界面点 **「加载」**（或重启）即读到新场景——**主机改一次、大家都更新，不用重新发程序**。

> 配比经验（一本 8–12 张）：full 2–3 / medium 3–4 / close 2–3 / closeup 1–2；同一场景复用时尽量换景别。

### 4.4 离线/便携场景（可选）
若某台机器上不了局域网：把整个 `wedding_scene_lib\` 拷到该机器本地任意目录，界面「场景库」指到本地那份、点「加载」即可（程序也支持本地路径）。

## 五、人物入景 · 生产流程（对齐 RunningHub 的产线手感）

| 步骤 | 操作 |
|---|---|
| 配置 | 调色 / 场景池 / 原片文件夹 / 输出文件夹 / **并发数(1–8)** / **跳过已处理(断点续跑)** |
| 开始 | 点 **「▶ 开始」**：并发 AI 判景别 → 选景装配 → 双图出图 |
| 看进度 | 进度条 + **✓成功 / ⏭跳过 / ✗失败** 计数；左侧列表可按 **全部/进行中/成功/跳过/失败** 筛选 |
| 纠错 | AI 判错景别？左侧单子上改景别（全身/中景/近景/特写）+ 单/双人 → 再点开始（只补没出的） |
| QC | 点单子看 **原片↔结果** 全分辨率对比 → **✓ 选用 / ✗ 判废** |
| 交付 | **「归集已选」**→ 精选片归档到 `输出\final\`；不够清晰的会标「需放大」，可切到 RunningHub 模式高清放大 |

**断点续跑**：输出按原片名 `原名_c.png` 命名，已有结果的图开始时直接「已跳过」、不重判不重出（省钱）。想全部重出 → 取消「跳过已处理」勾选。

> 典型一条龙：人物入景出图（约 1.6MP）→ 不够 2K 的，切到 RunningHub 模式做高清放大。

## 八、常见问题

| 现象 | 处理 |
|---|---|
| 双击弹「Windows 已保护你的电脑」 | 「更多信息 → 仍要运行」（未签名小程序通病） |
| 人物入景显示「⚠ 未加载」 | 访问不到局域网共享场景库 → 在文件资源管理器试开 `\\DESKTOP-66773HC\ai_work`（需要时输主机账号密码）；确认「场景库」路径是 `\\DESKTOP-66773HC\ai_work\wedding_scene_lib`，点「加载」 |
| 余额显示报错 / 查不到 | 检查 key.txt 第2行系统令牌、第3行用户ID（sk- 密钥查不了额度，必须系统令牌+用户ID） |
| 出图失败、日志报 503/网络 | 4sapi 渠道波动或断网，稍后重试；持久失败看余额/渠道 |
| 设置/配置记不住 | 别放 `Program Files`，换到可写目录（桌面/D盘） |
| 中文显示成方块 | 极少见，系统装了中文字体即可（程序自动找微软雅黑/黑体/宋体） |
| 脸/景别偶尔不对 | image2 有方差：改景别重出、或加候选；背身/远景小脸仍可能漂（需人工兜底） |
| 隐私 | 客户人脸会发第三方中转（4sapi），对外收费前确认客户授权 |

## 九、怎样把软件打包发给同事（给分发方）

### 9.1 打包清单（只三样，**不含场景库**）
1. `target\release\视觉海岸批量处理工作台.exe`
2. `key.txt`（凭据；共用账号就用现成的，或让同事填自己的）
3. `doc\使用文档.md`（本文件）

> 场景库不再进包——它在局域网共享 `\\DESKTOP-66773HC\ai_work\wedding_scene_lib`，程序默认就读那里。所以**包很小、永远不会越打越大**；扩库只在共享上改一次，全员生效。前提是同事电脑能访问该共享。

### 9.2 一键打包成 zip（PowerShell，分发方在项目根跑）
```powershell
# 版本号自动取自 Cargo.toml（迭代时改 Cargo.toml 的 version 即可，文件名/标题自动跟上）
$root = "D:\xc_test\new_proj\rust_prjo"
$ver  = ((Select-String -Path "$root\Cargo.toml" -Pattern '^version\s*=\s*"(.+)"').Matches.Groups[1].Value)
$dist = "$env:USERPROFILE\Desktop\视觉海岸批量处理工作台_v$ver"
Remove-Item $dist,"$dist.zip" -Recurse -Force -ErrorAction SilentlyContinue
New-Item -ItemType Directory -Force $dist | Out-Null
Copy-Item "$root\target\release\视觉海岸批量处理工作台.exe" $dist
Copy-Item "$root\key.txt" $dist
Copy-Item "$root\doc\使用文档.md" $dist
Compress-Archive -Path "$dist\*" -DestinationPath "$dist.zip" -Force
Write-Host "✅ 打好了：$dist.zip"
```
把生成的 `视觉海岸批量处理工作台_v<版本>.zip`（只几 MB）发给同事即可。**界面右上角显示版本号**，方便核对大家用的是不是同一版。

### 9.3 注意事项
- **可写目录**：告诉同事解压到桌面/D盘，别放 Program Files。
- **局域网**：同事电脑要能访问 `\\DESKTOP-66773HC\ai_work` 共享（场景库在那里）；首次访问可能要输主机账号密码。
- **凭据安全**：zip 里含 `key.txt`（密钥），只走可信渠道发；公开分享请去掉 key.txt、让同事自己填。
- **SmartScreen**：未签名 exe 首次运行会拦，让同事「更多信息→仍要运行」。要彻底免拦需买代码签名证书（个人分发一般不必）。
- **改了代码/场景后重发**：重新 `build.ps1 -Release` 出新 exe（先关掉同事/自己正在运行的旧程序，否则覆盖不了），再跑 9.2 重新打包。
- 更详细的安装包（Inno Setup setup.exe / 开始菜单快捷方式）见 `doc\打包与安装指南.md`。

## 十、一句话总结

把 **exe + `key.txt`** 放在同一个**可写文件夹**、能上局域网 → 双击 exe → 切到「人物入景」→ 填原片/输出文件夹、选调色 → **「▶ 开始」**，结果自动出到输出夹。**场景库在共享 `\\DESKTOP-66773HC\ai_work\wedding_scene_lib`，扩库只在共享上改一次、全员生效，程序包不再带场景库、永远很小。**
