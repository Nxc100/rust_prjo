## 0. 一句话定位

一个 **Windows 桌面 GUI 程序**（纯 Rust，单文件 exe，零运行时依赖），给婚纱影楼修图师做**批量 AI 出图**。当前正从「单一能力」扩展为「**多能力工作台**」：

- **能力一（已成熟）**：RunningHub 批量**精修 / 高清放大**——把客户原片成批跑 ComfyUI 工作流。
- **能力二（整合中）**：**人物入景**——把真人原片放进固定场景库，自动判景别→选景→统一调色，用 AI 出成套写实婚纱样片。

两者将共用同一套界面外壳（图片列表、前后对比、进度、日志、账户），并可串成一条龙：人物入景出图 → RunningHub 高清放大。

## 2. 使用的模型

本程序**自身不含 AI 模型**，而是调用云端服务出图。涉及两个后端、两类模型：

| 能力 | 后端 / 接口 | 模型 | 说明 |
|---|---|---|---|
| **精修 / 高清放大** | RunningHub OpenAPI（`https://www.runninghub.cn`） | 由用户选定的 **ComfyUI 工作流**决定 | 示例工作流为 **Flux2-Klein 人像精修 + 高清放大**；程序自动识别工作流的 `LoadImage`/`SaveImage` 节点与可调参数（KSampler 种子/步数/CFG/重绘、CLIPTextEncode 提示词、ImageScaleBy 放大倍数） |
| **人物入景 · 出图** | 4sapi（`https://4sapi.com`），端点 `/v1/images/edits` | **`gpt-image-2`** | OpenAI 系图像编辑模型，经 4sapi 中转（Codex 上游）。**双图编辑**：图A人物 + 图B场景 plate，按装配好的提示词重绘合成 |
| **人物入景 · 景别判定** | 4sapi，端点 `/v1/messages`（Claude 原生格式，同令牌） | **`claude-sonnet-4-6`** | **已接入一键流**：点「开始」自动逐张判景别+人数（`foursapi::judge_shot`，缩略图 base64 上传，结构化 JSON 输出）；操作员可在左侧手改后重跑。AI 预筛 QC 仍待做（同模型） |

> 模型选择原则：构建 AI 能力时优先用最新最强模型；当前出图锁定 `gpt-image-2`（同事系统实测此模型出图协调、可双图锁脸）。

## 3. 技术栈

- **语言**：Rust 2021，工具链 `stable-x86_64-pc-windows-gnu`（GNU 工具链，自带链接器，无需 MSVC / Windows SDK）。
- **GUI**：`eframe` / `egui` 0.34（立即模式 GUI，默认 wgpu/DX12 渲染）。
- **网络**：`reqwest` 0.12（blocking + multipart + json，TLS 走系统 Schannel `default-tls`，不依赖 OpenSSL）。
- **其它**：`serde`/`serde_json`（配置与接口）、`rfd`（原生文件对话框）、`crossbeam-channel`（后台→UI 消息）、`image`（纯 Rust 缩略图解码）、`base64`（解码 4sapi b64 出图）、`anyhow`（错误）。
- **构建期**：`winresource`（嵌入 exe 图标）。
- **测试**：`egui_kittest`（无头模拟真实控件树点击/渲染）。
- **产物**：单个自包含 `.exe`（约 9.3 MB），Win10/11 双击即用，**零运行时依赖**（无 VC++/MinGW/.NET/WebView2/OpenSSL）。

```
│   ├── workflow.rs               解析 ComfyUI 工作流 JSON，识别输入/输出节点 + 可调参数
│   ├── cmode.rs       【新】      人物入景「大脑」：选景 + 四轴提示词装配（移植 match_scene.py）
│   ├── foursapi.rs    【新】      4sapi gpt-image-2 双图出图客户端

├── assets/wedding/    【新】      人物入景的可热插拔数据（含 18 张 plate jpg）
│   ├── templates/人物入景_双图暖金版.md    出图提示词模板（机器可读区块）
│   ├── scenes/catalog.json                场景库目录（18 张 plate 的标签/构图）
│   ├── scenes/series_profiles.json        场景配置卡（四轴拆开的“场景专属话”）
│   └── _golden/                           与 Python 对拍的黄金样本（测试用）
├── doc/                          实现方案、打包指南、本文档、原始 rh_batch.py、样片
└── tips/                         4sapi 接口文档（gpt-image-2 / edits / 图片理解 / claude）
```

**各模块要点**

- **`main.rs`**：三栏布局（顶部控制+进度+设置 / 左侧列表 / 中间对比 / 底部日志）。含可持久化配置（预设 `Preset` + 窗口布局 `Store`，存 exe 同目录 `vc_batch_config.json`）、后台线程池跑批、对比查看器（并排/滑动、缩放平移、全分辨率）、清单 CSV。**顶部 `AppMode` 模式切换**：RunningHub 与人物入景共用同一外壳，各自的头部/列表/预览按模式分支（`w_header`/`w_list`/`w_preview` + 背景出图线程）。
- **`api.rs`**：`RhClient` —— `upload_image` / `create_task`（解析队列占满、节点校验提示）/ `poll_once`（完成/排队/不确定三态）/ `download` / `account_status`（余额、并发占用）。含网络重试退避。
- **`workflow.rs`**：`detect_io_nodes`（唯一 LoadImage→输入、唯一 SaveImage→输出）+ `detect_params`（从 widgets_values 提取常见可调参数，做成表单）。
- **`cmode.rs`【新】**：`SceneEntry`/`PersonAttrs` 数据结构；`load_from_dir` 加载模板/目录/配置卡；`can_serve`/`rank_candidates`（景别层级：宽景别图可借给窄景别）；`auto_pick_scene`/`attr_score`（最配优先 + 复用排重）；`assemble_prompt`（**四轴正交装配**：景别×调色×场景概念×plate）；`size_for_aspect`（按原片宽高比选出图尺寸）。
- **`foursapi.rs`【新】**：`FsClient::edit_dual`（双图 multipart 上传出图）；响应兼容 `b64_json` 与 `url` 两种回传；**瞬时错重试 / 持久错响亮中止**（纯 API 命门）。
- **`cstate.rs`【新】**：`CManifest`/`CJob`/`CStatus`（pending→prompted→awaiting_qc→selected→done｜qc_fail）；`init`（扫文件夹建账）、`set_job`（改计划字段+失效回退：宽→窄保留场景、窄→宽清空重选）、`auto_plan`（自动选景排重）、`assemble`（调 cmode 装配+复用即变构图）、`select`/`fail`/`collect`（选片+需放大标记+归集）、JSON 持久化。

## 5. 功能清单

### 5.1 RunningHub 批量精修 / 高清放大（现有，可用）

- **配置表单**：API Key、工作流 ID、输入/输出文件夹、工作流 JSON、并发数（1–8）。
- **配置预设**：多套参数命名保存、切换、增删改（如「精修」「4K 放大」各一套）。
- **拖拽导入**：拖文件夹设输入目录、拖图片追加处理。
- **工作流智能识别**：读 ComfyUI 工作流 JSON 自动识别输入/输出节点；**参数表单化**（种子/步数/CFG/重绘/提示词/放大倍数勾选覆盖）。
- **批量执行**：后台线程池并发、断点续跑（已有 `原名_rh*` 结果自动跳过）、停止、失败重试 / 重跑选中。
- **逐图进度**：等待/上传/提交/生成/下载/完成/跳过/失败八阶段，状态徽章 + 列表筛选 + 跟随处理中。
- **前后对比查看器**：并排 / 滑动对比，缩放平移、1:1、**全分辨率**（刚修复，放大不发虚）。
- **账户与预警**：常驻余额/并发徽章、测试连接、跑前余额预警（按每张预估消耗）。
- **实时日志**：级别过滤 + 搜索 + 导出。
- **收尾**：完成横幅汇总、清单 `_manifest.csv`、任务栏闪烁 + 提示音 + 自动打开输出夹。

### 5.2 人物入景（L1 已可用 · 顶部模式切换进入）

- **四轴解耦装配**（M1 ✅）：景别(SHOTS) × 调色(KEY：自然/暖/冷) × 场景概念(配置卡) × 具体 plate(comp_prompt)。加新风格只改数据、不改代码。
- **景别层级选景**（M1 ✅）：宽景别场景图可借给窄景别复用，像「同一次拍摄」；自动选景 + 复用排重 + 复用即变构图。
- **双图出图**（M2 ✅）：图A人物（锁脸/锁姿势/锁衣饰/锁头部朝向）+ 图B场景 plate，4sapi `gpt-image-2` 重绘合成。
- **状态机**（M3 ✅）：客户 manifest、断点续跑、选景排重、改景别失效回退、QC 流转、归集 + 需放大标记。
- **模式界面 · 一键流**（M4 ✅）：顶部切到「人物入景」→ 配置（4sapi Key/调色/场景池/原片·输出文件夹/场景库）→ **一个「▶ 开始」**。后台自动：**AI 判景别+人数（`claude-sonnet-4-6` 视觉）→ 自动选景排重 → 装配 → 双图出图 → 落到输出夹**（已存在结果自动跳过＝断点续跑，可「■ 停止」）。左侧仍可逐单手改景别/人数后重跑；中间原片↔候选全分辨率对比 + 可选 QC 选用/判废/归集。与 RunningHub 模式同样「填文件夹→点开始」的极简体验。
- **测试连接 + 实时额度**（✅）：`foursapi::account()` 调 `GET /api/user/self`，「测试/刷新」按钮后台查 4sapi 余额并显示「🪙 余额 ≈ $X」。**注意**：该面板接口需 4sapi**系统访问令牌**（个人设置页生成，非出图用的 sk- 密钥）+ 可选用户 ID；故界面单列一个「访问令牌」字段（可留空，留空则用 sk- 查、多半被拒并提示）。出图结束自动刷新额度。
- **待做**：串接 RunningHub 放大（M5）；AI 预筛 QC（L2 剩余项）；C 模式配置持久化 + 滑动对比。

```

        →(可选)送 RunningHub 高清放大
```

## 9. 已知约束与风险

**RunningHub 侧**
- 并发数受 apiKey 套餐额度限制；基础套餐填 1，否则频繁「队列占满」。
- 放大任务慢（单张数分钟，超时默认 1800s）。

**人物入景 / 纯 API 侧**（来自同事交接文档）
- **纯 API 无回退**：4sapi 掉线 / 渠道无货（503）/ 余额 0 → 整条线停。建议留备用中转做 API 级回退。
- **共用令牌、余额共享**：跑干整批失败，要有人盯余额、及时充。
- **隐私/合规**：客户人脸照会发第三方中转，对外收费前需想清授权与数据处理。
- **交付分辨率**：默认 1.6MP 不放大；要 2K+ 才人工 Topaz（或本工作台 RunningHub 放大兜底）。放大会把小手锐成畸形、需局部修手。
- **质量边界**：背身/小脸/远景脸仍可能漂 → 走人工贴真脸（B 路线，系统不自动）；plate 必须无水印；全册基调一次定死、别中途换。

**工程侧**
- 人物入景引擎已就绪但**未接 UI**，终端用户暂用不到。
- 预览大图为控显存只驻留当前选中项，来回切换大图会按需重解码（超大图零点几秒延迟）。

## 12. 同事系统全貌复看 · 每个部件的自动化归宿

「人物入景」原系统由「编排层（算法）+ AI（出图/判图）+ 人工（QC）」三段构成。下面把它**所有**部件逐一映射到本工作台里的去向——这就是整合的全集。

| 原部件（同事 skill） | 职责 | 在本工作台里 |
|---|---|---|
| `prep_input.py` | 把客户原片拷进 input | ❌ 不移植——UI 直接「选客户文件夹 / 拖拽」（外壳已有） |
| `c_batch.py` 状态机 | manifest 记账/续跑/选景排重/QC流转 | → **M3 原生重写**（复用工作台 store/清单） |

## 13. 高度自动化设计

### 13.1 自动化的本质：识别并收敛「人工触点」

整条「人物入景」流水线只有**两个**不可省的人工判断，其余皆可自动化：

```
选客户文件夹 →〔判景别+人数〕→ 自动选景 → 自动装配 → 自动出图 →〔QC 选片〕→ 自动归集 → 自动放大 → 拼样片
              ▲ 人工触点①                                        ▲ 人工触点②
```

- **触点①「判景别」**：交接文档 §4.2 明确**允许用视觉 API 自动判**（判错不致命、可手改）→ 应自动化。
- **触点②「QC 选片」**：铁律「验收不接 AI，脸漂/手畸形/调色由人终审」→ **最终裁决保留给人**，但可用 AI **预筛标红**把人从「逐张全看」降为「只看 AI 标出的疑点 + 终审选片」。

把这两点收敛好，系统就从「人盯着跑」变成「人只在该决策处出现」。

### 13.2 端到端目标流水线（操作员视角）

```
1) 新建客户 → 选输入文件夹（拖拽即可）
2) 一键「智能判图」：AI 逐张判 景别/人数/person_attrs（低置信标黄让人复核，可手改）
3) 选全册基调（自然/暖/冷）+ 场景概念池（步步生花/礁石）
4) 一键「排片」：自动选景 + 排重 + 配比检查（full 2-3/medium 3-4/close 2-3/closeup 1-2），覆盖缺口报警
5) 一键「全册出图」：批量双图出图（失败自动重试/降档策略），实时进度
6) QC：每张「原片↔候选」全分辨率对比 + AI 预筛标红 11 项疑点 → 人终审 select / 重出
7) 一键「归集」：选片归档 + 标记需放大者 → 一键送 RunningHub 高清放大 → 拼样片交付
```

### 13.3 自动化分级（务实演进，不一步到位）

| 级别 | 形态 | 对应里程碑 |
|---|---|---|
| **L1**（v1） | 手动判景别 → 自动选景/装配/出图 → 人工逐张 QC | M3 + M4 |
| **L2** | **AI 判景别**(可改) → 批量出图 → **AI 预筛 QC** + 人终审 | M4 增强 + 智能判图/预筛 |
| **L3** | 全册**一键**（判→排片→出图→预筛）→ 人只终审选片 → 一键归集+放大+拼样 + 失败自动重出 | M5 + 自动重出策略 |
| **L4** | 闭环学习：用 `picks.jsonl` 人工选片校准选景权重/判景别；成本与质量看板 | 后续 |

> 推进顺序：**L1 已跑通**（M1–M4 全部完成、模式界面可用）。接下来叠加 L2 的两个 AI 升级（判景别 / 预筛 QC，模型 `claude-sonnet-4-6`），再做 M5 串接放大与 L3/L4。

### 13.4 两个核心 AI 自动化升级（设计要点）

**① AI 判景别 + person_attrs（触点①）**
- 调 4sapi `/v1/messages`（Claude 原生格式），**模型用 `claude-sonnet-4-6`**（同令牌，文档见 `tips/图片理解`），输入人物原片，**结构化输出** `{shot, subjects, facing, open_space, has_train, pose_density, view_angle, mood, confidence{...}}`。
- **提示词直接来自《景别判定与QC手册》§一/§七**——四档景别线索 + 8 字段 person_attrs 取值表，把"谁判都一样"的人类规则变成模型的判定标准。
- 兜底：低置信项标黄、操作员可一键改；判错由 QC 兜底（与手册同口径）。

**② AI 预筛 QC（触点②，严守"终审在人"边界）**
- 对每张候选，调视觉模型（同样 `claude-sonnet-4-6`，`/v1/messages`）按 **11 项 2C 评分单**（保本人/姿势/服装/景别占比/接地/基调/双人/手部/干净 + 比例透视/空侧无冲突）与**原片比对**，输出每项 ✓/✗/存疑 + 简述。
- **AI 只标红、不裁决**：把人从"逐张全看"降为"先看 AI 标✗的项再终审"。最终 select/重出仍由人按铁律决定。
- 边界纪律：这是**辅助预筛**，不是"AI 验收"；上线前需与负责人确认（铁律5），且预筛结论始终可被人推翻。

### 13.5 工程自动化：把 selftest 变成 Rust 回归门禁

同事用 `selftest.py` 在每次改数据/模板后离线自检。本工作台应有等价的 **Rust 回归测试**作为每次构建的门禁——部分**已落地**：
- ✅ **装配矩阵对拍**：`cmode` 的 `prompt_parity`/`rank_parity` 已覆盖「4 景别×3 基调×单双人×两 series 无残留占位符 + 与 Python 逐字一致」。
- ⏳ 待补（移植 selftest 剩余项）：catalog 数据完整性（id 唯一 / plate 文件在 / `comp_prompt` 无光照天气词 / `figure_anchor` 数值合法 / `series` 非空）、四轴解耦探针（新概念不漏旧场景词）、状态机干跑（init→set→plan→prompt→失效回退）。

这样同事按 §9 推送数据/模板更新后，跑一遍 `cargo test` 即可确认未退化——把"契约不变、随便更新、你无感"真正落到工程上。
